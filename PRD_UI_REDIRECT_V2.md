# PRD · Company Dashboard 独立站 UI/UX 重做（V2）

> 角色设定：Ninjin 为甲方；我方（NinjinOS）以“CEO→总监制”交付。
> 目标：先把“脸”做好看（半成品验收），再逐步补齐数据联动与自动化。

---

## 0. 甲方确认（已定）

- **北极星目标**：让人“读内容”（内容即产品）
- **首页形态**：杂志首页（栏目区块：文章/快报/视频/归档），克制、低噪音
- **核心内容**：Deepdive 长文
- **视觉系统**：瑞士网格（理性、秩序、出版物）
- **交付策略**：先交付半成品（视觉系统 + 首页 + deepdive 列表 + 文章页模板）

---

## 1. 问题陈述（Why redesign）

- 当前首页 UI/UX 被甲方判定为“不喜欢/难看”。
- 方向必须回到：**出版物式阅读体验**，而非“工具面板/信息墙”。

---

## 2. 成功标准（Definition of Done）

### 2.1 体验目标
- 首页第一眼像“编辑部首页”：结构清晰、留白克制、信息层级干净
- Deepdive 列表像 jimi.ink：日期→标题→摘要→标签，细分割线，舒服
- 文章页阅读舒适：宽度、行高、标题层级、引用/代码样式合理

### 2.2 验收页面（半成品必须交付）
- `index.html`（首页）
- `deepdive/index.html`（列表）
- `deepdive/posts/hello-deepdive.html`（文章模板）

### 2.3 约束
- 仍然是静态 HTML（不迁移框架）
- 不引入付费/登录依赖
- 避免 emoji 作为 UI icon
- 无障碍：对比度、focus、键盘可用
- 链接在 file:// 本地打开与部署后都能工作（相对链接优先）

---

## 3. 信息架构（IA）

### 3.1 顶部导航（简洁）
- 文章（Deepdive）
- 快报（Brief / Radar）
- 视频（Video/Podcast）
- 归档（Archive）
- 关于（About）

### 3.2 首页栏目（杂志结构）
1) **Hero**：一句主标题 + 一句副标题 + 1 个主 CTA（阅读最新 Deepdive）
2) **最新 Deepdive**（核心）
3) **今日快报 / 雷达**（简短列表/摘要）
4) **视频 / 播客**（轻量模块）
5) **归档入口**（搜索/标签）

---

## 4. 视觉系统（Swiss grid + warm paper）

- 背景：warm paper（低对比、不刺眼）
- 网格：12 栏或 8 栏网格，清晰的 gutter
- 分割线：薄、淡（替代卡片阴影堆叠）
- 字体：优先“出版物感”字体对（可选）：
  - 方案 A：Libre Bodoni（标题）+ Public Sans（正文）
  - 方案 B：保留 Fraunces + Space Grotesk（如果更贴合现有品牌）

> 具体由设计总监在实现阶段做 A/B 对比并择优。
